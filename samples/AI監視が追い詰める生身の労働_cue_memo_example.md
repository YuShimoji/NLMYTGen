# Cue Memo Example

source: `AI監視が追い詰める生身の労働.txt`  
packet: `AI監視が追い詰める生身の労働_cue_packet.md`  
note: B-15 Phase 1 の output contract 例。最終演出判断そのものではなく、S-6 の下調べメモ用。

## Summary

配送ドライバーと倉庫労働者をめぐる AI 監視の問題を、身体的限界の無視、路上へのリスク外部化、Amazon の PR 戦略と責任回避、そして将来の人間排除という問いへ段階的に広げていく構成。映像面では「密閉された監視空間」から「都市空間の危険」へ、最後に「未来の分岐点」へ視野を広げる流れが合う。

## Sections

### S1

- topic: AI 監視が人間の切実な事情を無視する導入と、倉庫内パノプティコンの説明
- start_index: 1
- end_index: 10
- primary_background: 猛暑の配送車内とダッシュボード周辺
- supporting_visual: 監視 UI または物流ネットワーク図のどちらか 1 つ
- emotion_cue: 導入は緊張と圧迫感、倉庫説明では冷たさと管理感を強める
- sound_cue_optional: 低めの持続音は合う。SE はロックオン描写など明確な箇所があるときだけ
- transition_cue: 車内の切迫した個人視点から、倉庫全景やシステム図へ引いていく
- operator_note: 「息苦しさ」「脇見運転フラグ」「ストップウォッチ化したスキャナー」が視覚フック。情報量が多いので、背景は細かく変えすぎず 2〜3 パターンで支える方が整理しやすい

### S2

- topic: タイム・オフ・タスク、精神的負荷、インセンティブによる危険運転誘導
- start_index: 11
- end_index: 13
- primary_background: タイマー表示や倉庫監視 UI
- supporting_visual: 炎天下の道路または焦るドライバー手元のどちらか
- emotion_cue: 圧迫感を維持しつつ、身体とアルゴリズムの衝突を強く見せる
- sound_cue_optional: テンポを少し上げる程度で十分。SE は必要なら場面転換だけ
- transition_cue: 倉庫内の管理から、街路や交通の危険へスケールを移す
- operator_note: 数値や倍率の説明が続くので、背景は「危険の増幅」を支える方向に寄せるとよい。過激な事故映像寄りより、監視と誘導の構造が見える素材の方が主題に合う

### S3

- topic: Amazon の PR と責任回避、労働組合の批判、最後の思考実験
- start_index: 14
- end_index: 28
- primary_background: アプリ UI や企業的配送イメージ
- supporting_visual: 規約文風テキストまたは未来的都市イメージのどちらか
- emotion_cue: 中盤は冷笑と批評性、終盤は問いかけと不穏さを強める
- sound_cue_optional: 終盤は余白のある音へ寄せる程度。SE は基本不要
- transition_cue: 「感謝の演出」と「責任回避」のズレを見せた後、未来の問いへ静かに広げる
- operator_note: ラストは断定より思考を促す締めなので、背景も過度に派手にせず、問いを残す余韻を優先した方が良い

## Global Notes

- 全体として「効率化の美名」と「生身の身体」の衝突が主題なので、背景は便利さの演出より監視・圧迫・外部化を支える方向が合う
- 表情は驚きと憤りを主軸にしつつ、終盤だけ少し思索寄りに落とすと流れが作りやすい
- データや制度説明の箇所は背景を頻繁に切り替えるより、要点の変わり目だけ押さえた方が理解しやすい
- sound cue は optional として扱い、映像判断の邪魔になる場合は省略してよい

## Operator Todos

- S1 の導入で使う「車内の切迫」と「監視 UI」の素材候補を先に 2〜3 点探す
- S2 は「暑さ」「タイマー」「道路リスク」を同じトーンで並べられる素材に絞る
- S3 の後半は PR 素材と未来的素材の落差が大きくなりすぎないよう、色味と明度を揃える
- 実際の YMM4 上では、重要な数字が出る箇所だけ背景切替か表情切替を入れ、それ以外はやりすぎない
