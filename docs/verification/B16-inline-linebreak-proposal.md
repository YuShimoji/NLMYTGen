# B-16 Proposal: 行内折り返し制御

## 何を解決するか

B-15 でページ間 (話者行) 分割は改善されたが、各話者行内の折り返しは YMM4 の自動折り返しに依存している。YMM4 は字幕表示枠の幅で機械的に折り返すため:

- カギ括弧ペアが行をまたぐ: `「エラー\nコード」`
- 単語途中で折り返す: `体\n調不良`, `スコ\nア`
- 数値+単位が分断: `71.4\n%`

## アプローチ: 外殻としての行内改行

現在の `reflow_utterance()` はページ間分割のみ。B-16 ではその出力の各チャンクに対して、**1行の最大文字数 (chars_per_line) で明示的に改行 `\n` を挿入**する。

```
B-15 (ページ間): 話者行テキスト → 複数の話者行に分割
B-16 (行内):     各話者行テキスト → 行内に \n を挿入して折り返し位置を制御
```

YMM4 は CSV 内の `\n` を尊重するため、明示改行があればそちらが優先される。

## 基本ルール

1. **1行の最大幅**: `chars_per_line` (display_width ベース)
2. **括弧ペアの保護**: `「...」` `（...）` の中身が `chars_per_line` 以内なら同一行に収める
3. **分割候補**: B-15 の大区切り (句読点, 閉じ括弧+助詞) を優先
4. **分割禁止**: カタカナ連続, 漢字連続, 数字+記号連続, 長音符前後
5. **候補がない場合**: 折り返さない (YMM4 自動折り返しに委ねる)

## B-15 との関係

```python
# B-15: ページ間分割 (話者行)
chunks = reflow_utterance(text, chars_per_line=40, max_lines=2)

# B-16: 各チャンク内で行内改行を挿入
for chunk in chunks:
    formatted = insert_inline_breaks(chunk, chars_per_line=40)
    # formatted には \n が含まれる
```

## 受け入れ条件 (草案)

| ID | 条件 |
|----|------|
| AC-1 | B-15 の既存テストを壊さない |
| AC-2 | カギ括弧ペアが chars_per_line 以内なら同一行に収まる |
| AC-3 | YMM4 取込後に「単語途中折り返し」が旧方式より減る |
| AC-4 | `--balance-lines` なしの既定動作は後方互換を維持 |

## リスク

| リスク | 軽減策 |
|--------|--------|
| 明示改行が YMM4 の表示と合わない | chars_per_line をユーザーが YMM4 テンプレートに合わせて設定 |
| 過剰な改行で行数が増える | max_lines で上限制御 |
| 括弧保護で1行が長くなりすぎる | chars_per_line 超過時は括弧外で分割 |
