# B-14 Proposal: aggressive clause chunking

## Why This Exists

B-13 後の post-import 再観測では、改善はあったがまだ次が残った。

- 手動改行: 5
- 再分割したい長文: 10
- 不自然な単語分割: 5

特に、長い一文が 1 字幕に残るケースがまだあり、operator からも「最初のいくつか以外は殆ど変わらないように見える」という評価が出た。

## Value Path

| 観点 | 狙い |
|---|---|
| どこに効くか | `--max-lines` 超過のまま残る単一長文 |
| 何を減らすか | YMM4 取込後の再分割と、1 字幕に残る長説明文 |
| なぜ今やるか | B-12/B-13 で軽い heuristics は一巡し、残 pain が「もっと積極的に複数字幕へ落とす必要があるか」に絞れたため |

## Proposal Scope

### やる

- `--max-lines` 超過の単一長文に対して、2 行字幕に無理に収めず、複数の字幕行へ chunking する fallback
- `、` / 接続句 / 引用句の区切りを使い、1 字幕あたりの意味まとまりを優先する
- `max_lines * chars_per_line` を少し超える段階で早めに chunk 候補を探す
- 長文 1 字幕が残るケースに限って、B-13 より強い分割を適用する

### やらない

- 本文の要約や削除
- GUI や YMM4 内自動調整
- 辞書登録の自動化
- LLM を使った意味解析

## Acceptance

| ID | 条件 |
|---|---|
| AC-1 | 既存 54 PASS を壊さない |
| AC-2 | 長い一文が複数字幕へ落ちるケースのテストを追加する |
| AC-3 | B-13 までで改善した手動改行件数を悪化させない |
| AC-4 | sample 再観測で「再分割したい長文」が 10 から明確に減る |
| AC-5 | 不自然な単語分割を悪化させないか、減少させる |

## Risks

| リスク | 内容 |
|---|---|
| 過分割 | 字幕数が増えすぎてテンポが細かくなりすぎる |
| 意味分断 | 強めの chunking で、1 字幕の意味まとまりが崩れる |
| テンポ変化 | YMM4 上の読みテンポが変わり、別の pain を生む可能性がある |

## Minimal Ask

この packet があれば、次の判断は 1 点だけでよい。

- `B-14 aggressive clause chunking を approved にするか`
