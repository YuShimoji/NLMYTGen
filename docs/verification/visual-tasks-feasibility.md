# 視覚系タスク Feasibility 整理

## 対象工程と現在の痛点

| 工程 | 手動作業 | 推定時間比重 | 現状の支援 |
|------|---------|-------------|-----------|
| S-6a 背景配置 | トピックに合う背景画像を探し、タイムラインに配置 | 高 | なし |
| S-6b 背景Animation | Ken Burns エフェクト (パン/ズーム) の方向・速度設定 | 中 | YMM4テンプレート |
| S-6c 表情切り替え | 発話内容に応じた立ち絵の表情選択 | 中 | YMM4テンプレート |
| S-6d BGM | 雰囲気に合うBGMの選定・配置 | 低〜中 | テンプレート固定 |
| S-8 サムネイル | キャッチコピー、背景、立ち絵の表情を組み合わせ | 高 | テンプレート手動 |

## Python 責務境界内でできること

| アプローチ | レイヤー | Python変更 | 説明 |
|-----------|---------|-----------|------|
| **背景キーワード生成** | L2 (テキスト) | なし (GUI LLM) | C-07 演出メモの一部。台本からセクション別の背景検索キーワードを生成 |
| **素材検索支援 CLI** | L1 (取得) | あり | Pixabay/Pexels 等の無料素材 API でキーワード検索し、URL 一覧をテキスト出力 |
| **サムネイルコピー生成** | L2 (テキスト) | なし (GUI LLM) | 台本要約からキャッチコピー候補をテキスト生成 |
| **メタデータ生成** | L2 (テキスト) | あり or なし | E-02。タイトル/説明/タグの叩き台 |

## Python 責務境界外 (禁止)

| アプローチ | 理由 | 根拠 |
|-----------|------|------|
| サムネイル画像生成 (PIL等) | 画像生成は Python 責務外 | D-01 rejected, INVARIANTS |
| YMM4 演出自動設定 | .ymmp 操作不可 | C-02〜C-05 rejected |
| 背景動画のレンダリング | 動画生成は YMM4 責務 | ADR-0003 |

## D-02 (背景動画取得自動化) 再審査チェックリスト

D-02 は quarantined (汚染バッチ由来)。通常候補に戻すには以下の全項目を整理する:

| # | 確認項目 | 状態 | 判断基準 |
|---|---------|------|---------|
| 1 | **取得元の特定** | 未整理 | どの素材サイト/API を使うか (Pixabay, Pexels, Unsplash 等) |
| 2 | **権利確認** | 未整理 | ライセンス形態 (CC0, Pixabay License 等)。商用利用可か。帰属表示は必要か |
| 3 | **取得と受け取りの分離** | 未整理 | 「URL 一覧をテキスト出力」(L1取得) と「画像をダウンロード」(L1取得) の境界。Python は URL 出力まで? ダウンロードも含む? |
| 4 | **YMM4 への受け渡し** | 未整理 | ダウンロードした画像を YMM4 でどう使うか (手動配置? フォルダ指定?) |
| 5 | **価値経路** | 未整理 | 「背景を探す時間」が実際にどれだけ減るか。C-07 の背景キーワード生成と組み合わせて初めて価値が出るか |
| 6 | **既存ワークフローとの整合** | 未整理 | 現在ユーザーが背景をどこから取得しているか (手動検索? ストック素材?) |

### 再審査の進め方

1. ユーザーに現在の背景素材調達フローを確認 (どこから? 何分かかる? 権利は?)
2. C-07 演出メモの背景キーワードが使えるか workflow proof で確認
3. 素材 API (Pixabay 等) の無料枠で検索が実用的か spike 検証
4. 上記3点の結果で D-02 を approved / hold / rejected に判定

## C-07 v1 proof 結果 (2026-04-01)

- 背景候補 (ストック素材キーワード): NG。同じ動画を使い回す。方向自体が違う
- 作業時間: OK。意思決定プロセス削減
- セクション分割: OK
- 「雰囲気: 知的」のような抽象形容は全く参考にならない

### 判明した実態 (スクリーンショットから)

**4つの演出パターン:**
| パターン | 用途 | 構成要素 |
|---------|------|---------|
| A. 茶番劇 | 人物の行動・心情を視覚化 | いらすとや等の人物 + 吹き出し + 表情 |
| B. 情報埋め込み | ファクト・数値の伝達 | 数値・地図・物資アイテムを画面配置 |
| C. 雰囲気演出 | 場面設定 | フリー素材写真 + 立ち絵 |
| D. 黒板型 | 概念整理 | 暗い背景にテキスト/リスト |

**最大ペイン:** 配置作業自体ではなく「何分から何分まで何を表示するか」のバランス判断 + 情報不足時の取材・素材集め

### 方向転換

- D-02 の方向: 素材API検索支援 → **演出判断支援 (4パターンベースの演出指示)**
- C-07 を v2 に改善: 背景キーワード → 演出アクション指示 (発話単位)
- Python 変更は不要。GUI LLM プロンプト改善で対応

## 推奨アプローチの優先順位 (更新)

1. **C-07 v2 の検証** -- 4パターン演出指示が使えるか確認
2. **v2 が有効なら Custom GPT / Claude Project に固定化** -- 毎回コピペを避ける
3. **サムネイルコピー生成** -- 同じ GUI LLM 方式で、台本からキャッチコピー候補を生成
4. **B-16 行内折り返し** -- 視覚系と並行して進められる

## start gate (更新)

C-07 v2 の検証完了後に、視覚系の次ステップを判断:
- v2 成功 → Custom GPT 固定化 + サムネイルコピーへ
- v2 部分成功 → パターン定義の微調整
- v2 失敗 → 手動ワークフロー継続
