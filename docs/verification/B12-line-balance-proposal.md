# B-12 Proposal: 行バランス重視の字幕分割

B-11 S-5 workflow proof の結果を受けた、次候補の proposal packet。

---

## 何を解決するか

現状の `split_long_utterances()` は文末（。！？!?）でしか分割しないため、

- 1 文が長いとそのまま残る
- 総表示幅は許容でも、2 行のバランスが崩れる
- 語中やカギカッコ前後で不自然な見え方になりやすい

という問題が残る。

B-11 の初回 proof では、読み間違い 0 / timing 問題 0 に対して、
手動改行・再分割したい長文が約 30 箇所で支配的だった。

---

## Value Path

| 質問 | 回答 |
|---|---|
| この出力はどこへ入るか | `build-csv` の出力品質改善として、YMM4 取込前の CSV に反映される |
| 何の手作業が減るか | S-5 での手動改行、長文の再分割判断 |
| GUI や外部手動転記は残るか | 残る。ただし主 pain の一つである字幕改行 bulk rework を減らす |

---

## 提案スコープ

### 含む

- 2 行字幕を想定した、行バランス重視の分割 heuristics
- 読点 `、`、句点 `。`、閉じカギ `」` などを候補にした任意分割
- 語中分割や極端な片寄り行を減らすための簡易スコアリング
- 既存の `--max-lines` / `--chars-per-line` と整合する opt-in 改善

### 含まない

- YMM4 内の自動改行や字幕配置制御
- GUI プレビュー
- 動画尺の伸縮やテンポ制御
- 辞書登録の自動化

---

## 変更イメージ

### 現状

- `max_length` 超過時でも、文末が無ければ 1 発話のまま保持
- 分割できても「2 行の見た目バランス」は考慮しない

### B-12 の方向

- 2 行想定時は、総表示幅だけでなく「各行の偏り」も見る
- `、` や `」` の直後を候補にして、自然な改行点を探す
- 語中分割やカタカナ語の破断は避ける

---

## 受け入れ条件（草案）

| ID | 条件 |
|---|---|
| AC-1 | 既存の文末分割テストを壊さない |
| AC-2 | opt-in で有効化され、既定動作は後方互換を保つ |
| AC-3 | 1 文長文に対して、読点や閉じカギを使った自然な 2 行分割が少なくとも 1 系統で可能になる |
| AC-4 | 語中分割の代表例（例: 「というこ」）を減らせる回帰テストを追加する |
| AC-5 | B-11 proof の代表 sample で、手動改行 / 再分割したい長文の件数が減る見込みを説明できる |

## リスク

| リスク | 内容 |
|---|---|
| テンプレート依存 | 実際の収まりは YMM4 テンプレート差に左右される |
| 過剰分割 | 読点ベースの分割を広げすぎると、発話が細切れになる |
| ルール肥大化 | 日本語の自然さ判定を過剰にやると仕様が膨らむ |

## やらないこと

- S-5 の pain を GUI 欲求へすり替えること
- 「5 分で短い」問題を、本文削減バグと即断すること

---

## 承認時の最小実装単位

1. 既存分割ロジックの後段に、行バランス改善の opt-in 分岐を追加
2. 代表的な 2〜3 パターンの回帰テストを追加
3. sample で pre-import 差分を再確認
