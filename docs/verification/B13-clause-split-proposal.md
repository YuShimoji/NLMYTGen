# B-13 Proposal: clause-aware split + widow/orphan guard

## Why This Exists

B-12 `--balance-lines` で `。` を起点にした自然改行は改善したが、post-import 再観測では次が残った。

- 手動改行: 10
- 再分割したい長文: 15
- 不自然な単語分割: 5

主 pain は「句読点が少ない一文が長く続くケース」と「1文字だけの最終行」で、単なる行バランス調整だけでは閉じないと分かった。

## Value Path

| 観点 | 狙い |
|---|---|
| どこに効くか | `build-csv` 出力のうち、2行字幕に入れたいが一文が長すぎるケース |
| 何を減らすか | YMM4 取込後の再分割と 1文字最終行の手直し |
| なぜ今やるか | B-12 の residual pain が具体化し、次の L2 改善軸が節分割と widow/orphan 回避に絞れたため |

## Proposal Scope

### やる

- 句点がない長文を `、` や接続句で節分割する heuristics
- `ということ`, `ですが`, `しかし`, `ので`, `ため`, `一方で` など、切れ目候補の優先順位づけ
- 1文字だけの最終行や極端に短い最終行を避ける widow/orphan guard
- 漢字連なり・カタカナ語・カギカッコ直後直前の不自然な分割を避ける penalty
- 既存の `--max-lines ... --balance-lines` の opt-in 改善として足す

### やらない

- 本文の要約や削除
- YMM4 内の見た目自動調整
- GUI 化
- 辞書登録の自動化

## Acceptance

| ID | 条件 |
|---|---|
| AC-1 | B-12 の既存テストを壊さない |
| AC-2 | 句点がない長文で、`、` や接続句を使った節分割のテストを追加する |
| AC-3 | 1文字最終行や極端に短い最終行を避けるテストを追加する |
| AC-4 | 3字以上の漢字連なり、カタカナ語、カギカッコでの不自然な折り返しを悪化させない |
| AC-5 | sample 再観測で、少なくとも「再分割したい長文」か「不自然な単語分割」のどちらかが明確に減る |

## Risks

| リスク | 内容 |
|---|---|
| 過剰分割 | 切れ目候補を増やしすぎると1話者の行数が増えすぎる |
| ルール肥大化 | 接続句ルールを増やしすぎると保守しづらくなる |
| テンプレート差 | YMM4 テンプレートによって最適位置が変わる可能性は残る |

- `B-13 clause-aware split + widow/orphan guard を approved にするか`
