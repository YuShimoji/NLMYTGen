# YMM4 テンプレート実測手順書

> 目的: YMM4 テンプレートファイルの形式・構造を実測し、三層責務構造の設計判断に必要な事実を確定する
> 作成: 2026-04-02
> Actor: ユーザー (YMM4 操作) + assistant (解析)

---

## 準備

- 既存の制作済みプロジェクト (.ymmp) を YMM4 で開く
- タイムラインにボイスアイテム、画像アイテムが存在する状態にする

---

## 操作A: テンプレートファイル形式 (P1, P2)

### A-1: 単一ボイスアイテムのテンプレート保存

1. タイムライン上のボイスアイテムを1つ右クリック
2. 「テンプレートとして保存」(または類似メニュー) を選択
3. 保存先フォルダ・ファイル名・拡張子を記録

### A-2: 単一画像アイテムのテンプレート保存

1. タイムライン上の画像アイテム (背景) を1つ右クリック
2. 同様にテンプレートとして保存

### A-3: 複数アイテムのバンドル保存

1. タイムラインでボイスアイテムと画像アイテムを複数選択 (Ctrl+クリック)
2. 右クリック → テンプレートとして保存できるか試す
3. **できた場合**: 保存する。**できなかった場合**: その旨を記録

### A-4: テンプレートのエクスポート

1. テンプレート管理画面を開く
2. エクスポートボタンがあれば押す
3. 保存先と形式を記録

### A-5: setting フォルダの確認

1. YMM4 のインストールフォルダ → `user/setting/` を開く
2. テンプレート関連のファイル/フォルダがあれば内容を確認
3. スクリーンショットまたはファイルリストを取得

### 渡すもの

- [x] A-1 の保存ファイル
- [x] A-2 の保存ファイル
- [x] A-3 の保存ファイル (できた場合)
- [x] A-4 のエクスポートファイル (できた場合)
- [x] A-5 の `user/setting/` フォルダ構造 (スクリーンショットでも可)

---

## 操作B: 表情アイテム (P3)

### B-1: 表情アイテムの自動作成

1. タイムライン上のボイスアイテムを右クリック
2. 「表情アイテムの自動作成」を選択 (v4.16.0.0 以降)
3. タイムラインに表情アイテムが追加されたことを確認

### B-2: 保存

1. プロジェクトを **別名で保存** (例: `measurement_face.ymmp`)

### 渡すもの

- [x] 表情アイテムを含む .ymmp ファイル

---

## 操作C: エフェクト付きアイテム (P4)

### C-1: エフェクト追加

1. タイムライン上の任意のアイテムを選択
2. 映像エフェクトを1つ追加 (拡大登場、フェードイン、振動など何でもよい)

### C-2: テンプレート保存

1. エフェクト付きアイテムをテンプレートとして保存

### C-3: プロジェクト保存

1. プロジェクトを **別名で保存** (例: `measurement_effect.ymmp`)

### 渡すもの

- [x] エフェクト付きテンプレートファイル
- [x] 対応する .ymmp ファイル

---

## 観察ポイント (操作中に気づいたことがあれば)

- テンプレート保存時のダイアログに表示される選択肢やオプション
- テンプレート管理画面の構造 (フォルダ分け、検索、ショートカット設定)
- 「テンプレートとして保存」メニューが出ないアイテム種別があるか
- 複数選択時の挙動の違い

---

## 解析の進め方 (assistant 側)

ファイルを受領したら以下の順で解析する:

1. ファイル形式判定 (JSON / バイナリ / XML 等)
2. JSON の場合: 構造ダンプ + `$type` キーの一覧化
3. ymmp 内の表情アイテム: `$type` でフィルタして構造抽出
4. VideoEffects 配列: `$type` ごとのパラメータ構造を記録
5. 結果を YMM4-AUTOMATION-RESEARCH.md sec7 の再分類に反映
