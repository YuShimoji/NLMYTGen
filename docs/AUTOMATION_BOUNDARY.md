# AUTOMATION BOUNDARY — 自動化の境界

NLMYTGen の自動化がどこで動作するかを明確にする。
「YMM4 の中で何をするか」と「YMM4 の外で何をするか」を混同しないためのドキュメント。

---

## 全体像

```
┌──────────────────────────────────────────────────────────────────┐
│                    動画制作ワークフロー                            │
│                                                                  │
│  [L1 入力取得]     [L2 変換]        [L3 YMM4内部]    [L4 配信]   │
│                                                                  │
│  NotebookLM   →  NLMYTGen CLI  →  YMM4           →  YouTube    │
│  RSS/ソース       Python           テンプレート       投稿API     │
│                   GUI (将来)        プロジェクト                   │
│                                     音声合成                      │
│                                     レンダリング                  │
└──────────────────────────────────────────────────────────────────┘
```

---

## レイヤー定義

### L1: 入力取得（NLMYTGen の外）

NotebookLM やその他のソースから素材を取得する段階。

| 何をするか | 誰がやるか | 自動化の余地 |
|-----------|-----------|------------|
| NotebookLM にソース投入 | 人間（手動） | RSS 連携でソース収集を半自動化 |
| Audio Overview 生成 | NotebookLM | API 未公開のため手動 |
| 元台本テキスト取得 | 人間（コピペ） | API 公開時に自動化 |
| 音声書き起こし（fallback） | Whisper / Google Docs | CLI で自動化可能 |
| 背景動画・素材の取得 | 人間 or スクリプト | 素材サイト API で自動化可能 |

**NLMYTGen がやること:** 取得されたテキストファイルを受け取る。取得自体はしない。
**将来:** RSS 連携、素材取得スクリプトを NLMYTGen に追加する可能性あり。

---

### L2: 変換（NLMYTGen の中 — Python）

テキストを YMM4 が読み込める形式に変換し、編集に必要な付随情報を生成する段階。

| 何をするか | 誰がやるか | 現状 |
|-----------|-----------|------|
| テキスト → YMM4 CSV | NLMYTGen CLI | done (build-csv) |
| 話者マッピング | NLMYTGen CLI | done (--speaker-map) |
| 長文分割 | NLMYTGen CLI | done (--max-length) |
| 入力検証 | NLMYTGen CLI | done (validate) |
| サムネイル生成 | NLMYTGen CLI (将来) | テンプレート + テキスト合成 |
| YouTube メタデータ生成 | NLMYTGen CLI (将来) | テンプレートベース |

**境界ルール:**
- Python は CSV / メタデータ / 画像テンプレート合成 までが限界
- **動画レンダリング、音声合成は Python ではやらない**
- 外部 TTS (Voicevox 等) は使用しない

---

### L3: YMM4 内部（NLMYTGen の外）

YMM4 上での操作。音声合成・字幕配置・動画レンダリングはすべてここ。

| 何をするか | 誰がやるか | 自動化の余地 |
|-----------|-----------|------------|
| CSV 台本読込 | YMM4（手動操作） | YMM4 のバッチ機能 / CLI があれば |
| 音声合成 | YMM4 内蔵 | YMM4 が台本読込時に自動実行 |
| 字幕配置 | YMM4 | 台本読込で自動配置 |
| 立ち絵・表情設定 | 人間 or YMM4テンプレート | テンプレートで定型化 |
| 背景・エフェクト配置 | 人間 or YMM4テンプレート | テンプレートで定型化 |
| タイムライン編集 | 人間 | テンプレートで初期配置を自動化 |
| 動画レンダリング | YMM4 | YMM4 が実行 |

**NLMYTGen がやること:** YMM4 が読み込めるファイル (CSV) を生成する。それ以上はしない。
**将来:** .ymmp (YMM4 プロジェクトファイル) を直接生成することで、演出情報を CSV より豊富に渡せる可能性あり。

**テンプレート戦略:**
YMM4 の演出を自動化する主な手段は「テンプレートの活用」。
- YMM4 プロジェクトテンプレート: 立ち絵・背景・レイアウトが設定済みのプロジェクト
- NLMYTGen が .ymmp を生成する場合、テンプレートの設定値を埋め込む
- 動的な演出（場面転換等）は、メタデータで「ここで背景を変える」等のヒントを渡し、YMM4 側で反映する

---

### L4: 出力・配信（NLMYTGen の外）

動画完成後の公開・配信。

| 何をするか | 誰がやるか | 自動化の余地 |
|-----------|-----------|------------|
| YouTube 投稿 | 人間（手動） | YouTube Data API v3 で自動化 |
| タイトル・説明・タグ設定 | 人間 | テンプレートで半自動化 |
| サムネイル設定 | 人間 | L2 で生成したサムネイルをアップロード |

---

## 絶対に越えない境界

| 禁止事項 | 理由 | 根拠 |
|---------|------|------|
| Python で動画をレンダリングする | YMM4 の責務 | ADR-0003 |
| Python で音声を合成する | YMM4 内蔵 TTS を使う | ADR-0003 |
| Voicevox / 外部 TTS を使う | 外部依存を増やさない | ユーザー指示 |
| Python で字幕を配置する | YMM4 の責務 | ADR-0003 |
| NotebookLM の代わりに LLM で台本を生成する | NotebookLM が upstream | ADR-0002 |
| ffmpeg で動画を合成する | YMM4 の責務 | ADR-0003 |

---

## 判断に迷ったとき

「この機能は Python でやるべきか、YMM4 でやるべきか？」の判断基準:

1. **それは「ファイルを生成する」作業か？** → Python (L2)
   - CSV、メタデータ JSON、サムネイル画像、.ymmp ファイル
2. **それは「音声・映像を合成する」作業か？** → YMM4 (L3)
   - 音声合成、字幕レンダリング、動画エンコード
3. **それは「YMM4 の設定を決める」作業か？** → テンプレート (L3) or メタデータ (L2)
   - テンプレートで定型化できるなら L3
   - 台本に依存する動的な設定なら L2 でメタデータを生成し L3 で反映
4. **それは「外部サービスと通信する」作業か？** → L1 or L4
   - 入力取得なら L1、配信なら L4
